CodeMate-report-1

기업협업 프로젝트 Dog-Mate

4명에서 팀을 이루어 프론트2명 백엔드 2명 으로 시작하였다.

프론트와 백엔드 스택은 이렇다 .

첫번째 2주 프로젝트와는 다르게 새로운 기술 스택이 추가되었다.

Front End React,Redux,Redux toolkit,material ui

Back End TypeORM,express,typescript-rest

Etc TypeScript,eslint,mocha/chai,prettier

1차 배포 후 피드백

배포를 진행하고 기업에서 일하고 계신분에게 피드백을 얻게 되었다.

프론트에서는 전반적으로 redux 를 쓰며 이익을 얻는 면이 부족한것 이 아닐까라는 의견을 듣게 되었고 저또한 코드를 작성하며 redux 를 굳이 써야하는 이유를 아직 깨닫지 못했다.

오히려 react를 쓴다면 코드가독성이랑 간편하게 작성할수 있을것 같다라는 느낌을 갖게 되었다. typescript 는 확실히 백엔드 쪽은 얻는 type을 정확히 쓰지않으면 어류를 발생시켜 잡아주는 역활을 하지만 프론트에서 typescript 는 오히려 코드를 작성하는데 까다롭게 제한을 받는 느낌을 받게 되었다.

1차 Sprint Review

Fact (객관적 사실)

프론트엔드를 맡아 리액트-리덕스를 사용해 페이지를 구성했다. 타입스크립트를 사용했다. 파일구조를 초기에 잡아두고 페이지를 작성할 때 전체적인 구조를 생각하면서 작업했다.

Feeling (느낌)

리덕스를 처음 적용할 때 작성해야 하는 부분이 많아서 번거로웠다. 타입스크립트를 프론트에 적용하는 것에 의문이 들었다. 구조를 나눠서 사용하니까 협업할 때 좀 더 편리했다. 여전히 기능 구현에만 집중해서 코드를 작성하는 경향이 있다.

프론트엔드가 ui를 바로바로 볼 수 있는 점은 좋지만 작성해야 할 코드가 많다고 느껴졌다. 2주 때 백을 해서 이번에는 프론트를 경험해볼 수 있는 좋은 기회를 만난 것 같다. 초반에는 서버랑 연결되는 부분을 테스트해볼 수가 없어서 코드를 작성할 때 답답했다. 디자인을 잘 못해서 약간 절망스러웠다.

Finding (교훈)

코드 리뷰시간에 아주 많은 것을 얻을 수 있었다. 리덕스를 사용하면서 프론트엔드 로직을 어떻게 나눌 수 있는지에 대해 알게 되서 굉장히 좋았다. 잘 몰랐는데 스트레스 받으면 짜증을 많이 내는 타입이었던 것 같다;; 처음 시작할 때 많은 부분을 합의하고 잘 나눠서 시작하면 일할 때 편하다. 너무 나 자신을 굴리면 성격이 더러워진다..

Future Action (행동)

미들웨어에 대해 알아보고 로직을 구분해서 작성하고 싶고, 리액트와 리덕스의 역할을 구분해서 프로젝트를 구성하고 싶다. 그리고 스트레스가 쌓이지 않게 잘 관리해야겠다. 내 문제 때문에 다른 사람들에게 피해를 주지 않게 스스로를 잘 다스려야할 것 같다.
